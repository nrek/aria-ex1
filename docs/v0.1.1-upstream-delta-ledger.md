# v0.1.1 Upstream Delta Ledger

Comparison of `aria-knowledge` additions from v2.8.4 through v2.10.6 against the `aria-ex1` v0.1.0 baseline. Each item is classified by execution-first philosophy.

**Decision key:**
- **IMPORT** — directly improves hook correctness, session stability, or scope-control for coding work
- **OPTIONAL** — useful but not essential for shipping; revisit in v1.2+
- **REJECT** — adds recurring ceremony, knowledge-lifecycle coupling, or token overhead without execution gains

---
