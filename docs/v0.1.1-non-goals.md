# v0.1.1 Non-Goals (Exclusions - explicit)

Items from `aria-knowledge` v2.8.4–v2.10.6 that will NOT enter `aria-ex1` v0.1.1. Documented so future maintainers don't re-evaluate settled decisions.

## Permanently Out of Scope for aria-ex1

These are structural product boundaries, not deferrals:

1. **Knowledge capture / backlogs / intake** — `/extract`, `/intake`, `/clip`, `/ask`, `/backlog`, `/wrapup`, `/stats`, `/context` commands and all associated templates (`intake/`, `logs/`, `index.md`).
2. **Audit cadences and SessionStart audit prompts** — `audit_trigger_threshold`, `audit_cadence_knowledge`, `audit_cadence_config`, tiered messaging, knowledge-audit-log parsing.
3. **Compaction hooks** — `PreCompact`, `PostCompact` hooks and pre-compact capture pipeline.
4. **TaskCreated hook** — `task-context-check.sh` and knowledge-index tag matching.
5. **SessionStart hook** — ex1 intentionally has no session-start injection. Rule 22 ordering discipline is enforced structurally via the deny mechanism (hook-side), not via session-start text injection.
6. **Knowledge folder protection** — ex1 has no `knowledge_folder` concept; the conditional protection logic in hooks is unnecessary.
7. **Ideas backlog and staleness surfacing** — `ideas-backlog.md`, `ideas_staleness_threshold_days`, `/context` ideas section.
8. **Project-specific knowledge tier** — `projects/` subfolder, `projects_enabled`, project-to-tag mapping, cross-project promotion.
9. **Slash-command aliases** — `/knowledge-audit`, `/config-audit` delegates.
10. **OVERVIEW.md philosophy narrative** — ex1 has its own README positioning.
11. **ADR infrastructure** — ex1 does not maintain an ADR folder; decisions are documented in CHANGELOG.

## Deferred to v1.2+ (Potentially Useful, Not Critical)

1. **Batch-manifest mechanism** — `kt_batch_begin/end/find_match/clear_stale` + jq dependency. Useful for multi-file plan execution; evaluate after v1.1 ships.
2. **Stack-aware cross-cutting candidates in /codemap** — Proposes Django/Next.js/Laravel/Expo-specific sections. Nice enrichment; not blocking.
3. **Model Recommendations in /help** — Per-skill model tier guidance. Low effort to add later.
4. **`kt_project_for_path()` helper** — Auto-detect project from CWD. Could be useful for auto-context but ex1 doesn't use it yet.
